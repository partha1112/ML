An end-to-end Machine Learning web application designed to analyze the sentiment of movie reviews. It leverages a **Deep Learning LSTM model** trained on the popular IMDB movie review dataset, serves the inference engine via a **FastAPI backend**, and provides an interactive, beautiful interface built with **Streamlit**.

## 🧠 Machine Learning Model & Training

The core classification model is a **Recurrent Neural Network (RNN)** utilizing **Long Short-Term Memory (LSTM)** layers designed to process text sequences sequentially.

### Neural Network Architecture

| Layer | Type | Configuration / Shape | Parameters & Purpose |
| :--- | :--- | :--- | :--- |
| **1** | **Embedding** | `input_dim=5000`, `output_dim=200` | Embeds the top 5,000 vocabulary words into a dense 200-dimensional space. |
| **2** | **LSTM** | `units=128`, `dropout=0.2`, `recurrent_dropout=0.2` | Captures long-range context and sequence dependencies in the reviews while preventing overfitting. |
| **3** | **Dense** | `units=1`, `activation="sigmoid"` | Standard output layer producing a continuous probability score between 0.0 (Negative) and 1.0 (Positive). |

### Preprocessing Pipeline
1. **Vocabulary Limitation**: The tokenizer retains only the top **5,000** most frequent words to avoid dimensionality explosion.
2. **Text-to-Sequence Conversion**: Raw review text is mapped to integers representing vocabulary indices.
3. **Padding**: Sequences are zero-padded to a maximum length of **200 words** (`maxlen=200`) using pre-padding to standardize batch shapes.

### Model Performance Summary
- **Dataset**: IMDB Movie Reviews (50,000 reviews, perfectly balanced: 25k positive, 25k negative).
- **Split**: 80% Training (40,000 reviews) and 20% Testing (10,000 reviews). A 20% validation split was used during model fit (32k actual training, 8k validation).
- **Training Epochs**: `5` epochs with `batch_size=4` using the `Adam` optimizer and `Binary Cross-Entropy` loss function.
- **Evaluation Metrics**:
  - **Validation Accuracy**: `~88.91%`
  - **Test Accuracy**: `~88.93%`
  - **Test Loss**: `0.3332`

## 🚀 Setup & Execution Guide

### Prerequisites
- **Python**: Version `3.12` or higher
- **UV Package Manager**: It is highly recommended to manage Python projects and virtual environments using [uv](https://github.com/astral-sh/uv).

### 1. Installation
Clone the repository and synchronize the environment dependencies:
```bash
# Sync and install dependencies specified in pyproject.toml
uv sync
```

### 2. Run the FastAPI Backend
Start the FastAPI server on port `8000`:
```bash
uv run uvicorn controller.classify_controller:api --reload
```
Once started, you can explore the interactive API docs at http://127.0.0.1:8000/docs.

### 3. Run the Streamlit Frontend Client
In a separate terminal, launch the Streamlit frontend:
```bash
uv run streamlit run client/app.py
```
This will spin up a beautiful web application (usually at http://localhost:8501) where you can input custom reviews and visualize model predictions.

## 🔌 API Documentation

### Classify Sentiment
Exposes a single HTTP GET endpoint to classify reviews.

- **URL**: `/classify`
- **Method**: `GET`
- **Query Parameter**: `review` (String - the text of the review to classify)
- **Response Format**: `application/json`

#### Example Request
```http
GET http://127.0.0.1:8000/classify?review=The%20acting%20was%20fantastic%20and%20the%20plot%20was%20mindbending!
```

#### Example Response
```json
{
  "sentiment": "Positive",
  "confidence": 0.94
}
```
